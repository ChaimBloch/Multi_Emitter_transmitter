NI-USRP Simple Streaming (USRP RIO Devices)

The NI-USRP Simple Streaming sample project demonstrates the following tasks with the USRP-2940/2942/2943/2944/2950/2952/2953/2954 (USRP RIO):

  • Streaming data from the input port of the USRP RIO device to the host VI
  • Streaming data from the host VI to the output port of the USRP RIO device

USRP RIO Device Connectors

Throughout this document, the term input port refers to the RX1 or RX2 connector and output port refers to the TX1 connector.

System Requirements

  • LabVIEW 2016 Full or Professional Development System.
  • NI USRP RIO Instrument Design Libraries.
  • (Optional) LabVIEW FPGA Module 2016 to view, open, and edit the included FPGA VI. If you do not have the LabVIEW FPGA Module installed, you can still run this sample project. However, the Project Explorer window shows only the host VIs.

Features

  • Configures an input and output data stream to and from the host, other FPGAs, or inside the FPGA itself.
  • Applies frequency shifting and decimation on the FPGA to reduce data sent to the host, which can prevent buffer overflow.
  • Supports both continuous and finite acquisitions and generations.

Overview

This sample project contains three host VIs, which run on the host computer, and one FPGA VI, which runs on the FPGA. Each host VI configures the device input and output ports and configures a data stream between the host VI and the USRP RIO.

The sample project performs the following tasks:

  • Streaming data from the host to the device
    1. The host VI writes data to the host-to-target DMA FIFO. You can see this FIFO in the Project Explorer window at My Computer»<Device Name>»Resources»FIFOs»tx <Channel Number>.host fifo.
    2. The FPGA VI reads from this DMA FIFO, processes the data, and sends it to the output port.
  • Streaming data from the device to the host
    1. The FPGA VI acquires data from the input port, processes the data, and writes it to the target-to-host DMA FIFO. You can see this FIFO in the Project Explorer window at My Computer»<Device Name>»Resources»FIFOs»rx <Channel Number>.host fifo.
    2. The host VI reads from the target-to-host DMA FIFO and displays the data on the host VI front panel.

The output stream uses a Start Trigger to begin generating data to the output port. This stream can operate in the following modes:

  • Continuous—The host streams data to the FPGA until you stop the application. If the host does not write data to the FPGA quickly enough, the DMA FIFO does not have enough data to be read, and the FPGA reports a buffer underflow.
  • Finite—The host periodically sends a block of samples to the FPGA. Specify the size of each block in the Number of Samples host VI front panel control. A block of data is read on the FPGA when the Start Trigger is asserted.

    Set the mode using the Generation Mode host VI front panel control.

The input stream uses a Start Trigger to begin streaming data acquired from the input port. This stream can operate in the following modes:

  • Continuous—The FPGA streams data to the host until you stop the application. If the host cannot accept data quickly enough, the DMA FIFO fills too quickly, and the FPGA reports a buffer overflow.
  • Finite—The FPGA periodically sends a block of samples to the host. Specify the size of each block in the Number of Samples host VI front panel control. A block of data is sent when the Start Trigger is asserted.

    Set the mode using the Acquisition Mode host VI front panel control.

Host VIs

You can view the host VIs in the Project Explorer window by expanding My Computer. The following host VIs are included:

  • Rx Streaming (Host) VI—Streams data from the input ports to the host VI.
  • Tx Streaming (Host) VI—Streams data from the host VI to the output ports.
  • Tx and Rx Streaming (Host) VI—Streams data from the input ports to the host VI and from the host VI to the output ports.

For more information about using and connecting multiple USRP RIO devices, refer to the NI-USRP Help or to the appropriate specifications for your device:

  • NI USRP-2954R Specifications

FPGA VIs

You can view the FPGA VI in the Project Explorer window by expanding My Computer. The following FPGA VI is included:

  • Streaming Xcvr (FPGA) VI—Enables sample generation using the Tx Streaming Engine and sample acquisition using the Rx Streaming Engine. Specify the number of samples to generate using the tx.samples to generate parameter. Specify the number of samples to acquire using the rx.samples to acquire parameter.

If you have NI USRP RIO support installed, you can open the FPGA VI by navigating to My Computer»<Device Name>»Streaming Xcvr (FPGA) VI. You can modify the FPGA VI or create one as needed for your application.

Running this Sample Project

  1. Ensure all hardware is set up and visible in Measurement & Automation Explorer (MAX). Record the resource names of all USRP RIO devices because you will need these names later.
  2. In the Project Explorer window, open a host VI, such as Rx Streaming (Host) VI.
  3. In the RIO Device pull-down menu, specify the resource name of the USRP RIO device that you identified in step 1.
  4. Click Run. The front panel displays the acquired/generated data.
  5. Click Stop.
  6. Experiment with different values for the front panel controls and click Run. You must restart the VI for new settings to take effect.
